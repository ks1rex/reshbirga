# TODO / отклонения от ТЗ

- `market_orders`/`market_services` в проекте не существуют — реальные таблицы
  `orders` и `listings`. Колонка `category` и эндпоинт категорий добавлены к ним.
- `GET /market/categories` → реализован как `GET /listings/categories`
  (нет отдельного market-роутера, переиспользован listings).
- `GET /api/profile/...` → без префикса `/api`, как и весь остальной API
  (`GET /profile/:id/public`, `/profile/:id/reviews`, `/profile/:id/services`,
  `GET /profile/leaderboard`).
- `gost_master` (10+ использований ГОСТ-калькулятора): счётчик `gost_uses`
  инкрементируется в `POST /gost/buy-tokens` — единственном бэкенд-эндпоинте,
  фиксирующем обращение к ГОСТ-сервису. Реальный калькулятор работает через
  внешний `GOST_BACKEND_URL` и не репортит использование обратно — если нужно
  считать каждый расчёт, а не только покупку токенов, GOST-сервис должен
  вызывать вебхук на бэкенд.
- `referrer` (3+ друга): засчитывается как 3+ приглашённых пользователя,
  у которых `referral_qualifying_deposits_count > 0` (т.е. принесли рефереру
  хотя бы один бонус), а не просто 3+ регистрации по ссылке.
- `early_bird`: "30 дней после запуска сайта" — нет конфигурации даты запуска,
  поэтому за дату запуска принята дата создания самого первого аккаунта
  (`MIN(profiles.created_at)`). Если есть точная дата запуска — захардкодить
  в `grant_early_bird()` вместо подзапроса.
- Начисление репутации (`addReputation`) не атомарно (read-then-update) —
  ок для некритичных очков, не для денег. См. `ponytail:`-комментарий в
  `backend/src/utils/reputation.js`.
